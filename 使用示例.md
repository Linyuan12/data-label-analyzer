# 数据标注分析器 - 使用示例

## 🎯 新增功能

**v2.0 新增【正确标注】列**：自动修正幻觉标注，仅保留问题中实际存在的要素值

## 📊 输出格式

| 列名 | 说明 | 示例 |
|------|------|------|
| 问题 | 原始问题 | `查询工程用钢部现货 2025 年上半年的销售金额` |
| 原抽取结果 | 原始标注 | `{"部门": "工程用钢部", "订货用户": "本田贸易"}` |
| 标注 | 0=正确，1=有误 | `1` |
| 说明 | 错误原因 | `幻觉标注：'订货用户' 的值 '本田贸易' 在问题中不存在` |
| **正确标注** | **自动修正后的标注** | `{"部门": "工程用钢部"}` |

## 📝 使用场景

### 场景 1：批量修正标注

```bash
# Step 1: 运行验证
python3 validate_annotations.py --input data.xlsx --output results.xlsx

# Step 2: 筛选错误样本
# 在 Excel 中筛选【标注】= 1 的行

# Step 3: 使用【正确标注】列替换原标注
# 复制【正确标注】列 → 选择性粘贴到【原抽取结果】列

# Step 4: 重新验证确认修正完成
```

### 场景 2：分析常见错误类型

```bash
# 在 Excel 中打开 results.xlsx
# 对【说明】列进行文本分析，找出高频错误模式
# 例如：
# - 某个要素类型经常幻觉
# - 某个标注人员错误率高
```

### 场景 3：训练数据清洗

```python
import pandas as pd

# 读取验证结果
df = pd.read_excel('results.xlsx')

# 只保留正确标注
clean_df = df[df['标注'] == 0].copy()

# 或使用修正后的标注
df[' cleaned_label'] = df.apply(
    lambda row: row['正确标注'] if row['标注'] == 1 else row['原抽取结果'],
    axis=1
)

# 导出清洗后的数据
df.to_excel('cleaned_data.xlsx', index=False)
```

## 🔍 错误类型

| 错误类型 | 说明 | 处理方式 |
|----------|------|----------|
| `hallucination` | 标注的值在问题中不存在 | 使用【正确标注】列替换 |
| `partial_match` | 值与问题部分匹配 | 人工确认是否需要修正 |
| `missing_query` | 问题字段为空 | 检查数据源 |

## 💡 最佳实践

1. **迭代修正**：每次修正后重新验证，直到准确率>95%
2. **批量处理**：先处理高频错误类型，再处理个别案例
3. **保留原始**：始终保留原始标注列，便于回溯
4. **定期审核**：每周运行一次验证，监控标注质量

## ❓ 常见问题

**Q: 【正确标注】列是 JSON 格式，如何使用？**

A: 在 Excel 中：
1. 复制【正确标注】列
2. 在文本编辑器中替换 `"` 为 `'`（如需要）
3. 或使用 Python 的 `json.loads()` 解析

**Q: 如何只查看错误样本？**

A: 三种方式：
- Excel 中筛选【标注】= 1
- 命令行：`jq 'select(.标注 == 1)' results.jsonl`
- Python: `df[df['标注'] == 1]`

**Q: 准确率低怎么办？**

A: 检查：
1. 标注规范是否清晰
2. 标注人员是否理解任务
3. 是否有系统性错误模式（如某个要素类型总是幻觉）
