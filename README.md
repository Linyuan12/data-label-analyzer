# 数据标注分析器 - 快速开始

## 🎯 用途

验证 **NL2SQL 标注** 质量，检测**幻觉标注**（问题中没有的要素值）。

**输入格式：** Excel（.xlsx）
**输出格式：** Excel（.xlsx），自动新增【标注】和【说明】两列

## 🚀 5 分钟上手

### 方式 1: 修改变量（推荐）

1. 打开 `scripts/validate_annotations.py`
2. 修改顶部的变量：

```python
# ==================== 配置区域 ====================
input_file = ".openclaw/workspace/数据分析 0311"
output_file = ".openclaw/workspace/数据分析 0311_验证结果"

# 字段名配置
query_field = "问题"           # 问题字段名
label_field = "原抽取结果"      # 标注字段名

# Excel 配置
excel_sheet_name = "标注验证结果"  # Excel 工作表名称
# ================================================
```

3. 直接运行：
```bash
python scripts/validate_annotations.py
```

**自动输出：** Excel 文件，包含原始列 + 【标注】+【说明】

### 方式 2: 命令行参数（可选）

```bash
python scripts/validate_annotations.py \
  --input 输入文件.xlsx \
  --output 输出文件.xlsx \
  --query-field 问题 \
  --label-field 原抽取结果
```

## 📊 Excel 格式说明

### 输入格式

| 问题 | 原抽取结果 |
|------|-----------|
| 查询工程用钢部现货 2025 年上半年的销售金额 | `{"部门": "工程用钢部", "时间": "2025 年上半年"}` |
| 查询张三的订单 | `{"客户": "张三", "订货用户": "李四"}` |

### 输出格式

自动新增两列：

| 问题 | 原抽取结果 | ...其他列... | 标注 | 说明 |
|------|-----------|-------------|------|------|
| 查询... | `{...}` | ... | 0 | |
| 查询... | `{...}` | ... | 1 | 幻觉标注：'订货用户' 的值 '李四' 在问题中不存在 |

**标注说明：**
- `标注=0`：正确，说明列为空
- `标注=1`：有误，说明列填写具体原因

## 💡 工作流

```
1. 准备 Excel 数据 → 2. 运行验证脚本 → 3. 打开结果 Excel
                                              ↓
    5. 重新验证 ← 4. 筛选标注=1 的行 → 人工修正
```

**Excel 操作技巧：**
1. 打开结果文件
2. 筛选 `标注` 列 = 1 → 查看所有错误
3. 查看 `说明` 列了解错误原因
4. 直接编辑修正
5. 保存后重新运行验证

## 🔧 常用操作

### 在 Excel 中筛选错误

1. 选中表头行
2. 点击"数据" → "筛选"
3. 点击 `标注` 列的下拉箭头
4. 只勾选 `1` → 显示所有错误样本

## ❓ 常见问题

**Q: 字段名不是"问题"和"原抽取结果"怎么办？**

修改脚本顶部的变量：
```python
query_field = "你的问题字段名"
label_field = "你的标注字段名"
```

**Q: Excel 导出失败？**

安装依赖：
```bash
pip install pandas openpyxl
```

**Q: 标注结果解析失败？**

支持两种格式：
- JSON: `{"时间": "2025 年"}`
- Python dict: `{'时间': '2025 年'}`

**Q: 准确率多少算合格？**

- 训练数据：建议>98%
- 测试数据：建议>99%
- 快速迭代：至少>95%
