# 测试示例 - 用户提供的具体案例

## 输入数据

```json
{
  "query": "查询工程用钢部现货 2025 年上半年各年的销售合同金额含税、销售额不含税分别是多少？",
  "label": {
    "时间": {"交货半年": "2025 年上半年"},
    "指标": ["销售合同金额含税", "销售额不含税"],
    "维度": ["交货半年"],
    "限制条件": {
      "部门": "工程用钢部",
      "订单属性": "现货",
      "订货用户": "本田贸易 (中国)有限公司"
    }
  }
}
```

## 验证结果

```json
{
  "query": "查询工程用钢部现货 2025 年上半年各年的销售合同金额含税、销售额不含税分别是多少？",
  "label": {...},
  "标注": 1,
  "说明": "幻觉标注：'维度' 的值 '交货半年' 在问题中不存在；幻觉标注：'限制条件。订货用户' 的值 '本田贸易 (中国)有限公司' 在问题中不存在"
}
```

## 分析

### 问题中有的要素 ✅
- 部门：工程用钢部 ✓
- 订单属性：现货 ✓
- 时间：2025 年上半年 ✓
- 指标：销售合同金额含税、销售额不含税 ✓

### 问题中没有的要素 ❌
- **维度：交货半年** → 问题中只说了"2025 年上半年"，没有明确说这是"交货半年"维度
- **订货用户：本田贸易 (中国) 有限公司** → 问题中完全没提到

### 结论

这条数据**标注有误**（标注=1），需要修正：
- 删除 `维度` 字段，或确认是否有默认规则
- 删除 `限制条件。订货用户` 字段

## 使用方法

```bash
python scripts/validate_annotations.py \
  --input your_data.jsonl \
  --output results.jsonl
```

输出自动包含【标注】和【说明】两列。
